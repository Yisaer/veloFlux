# Documentation Index

This directory is organized by domain instead of by document creation time.

Use these top-level sections:

- `api/`: Runtime HTTP and introspection contracts.
- `syntax/`: StreamDialect SQL surface syntax and SQL-visible semantics.
- `planner/`: Planner behavior, optimization rules, and planner performance notes.
- `runtime/`: Runtime architecture, processors, source/sink execution behavior, and lifecycle.
- `integrations/`: External integration designs, including agent-facing workflows.
- `operations/`: Deployment, configuration, and CI/benchmark operational notes.
- `tests/`: Test design, coverage conventions, and test-maintenance specifications.

When adding new syntax-related documents, place them under `docs/syntax/` instead of the root.
